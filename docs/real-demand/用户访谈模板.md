# 用户访谈模板

日期：`2026-04-20`

这份模板用于访谈真实用户，验证 deployment / hardening 方向是否真的成立。

规则：

- 只问最近一次真实经历
- 不问抽象态度
- 不先推销方案
- 优先追问行为、代价、替代方案、AI 使用情况
- 必须追问为什么现有替代方案不够，以及什么情况下用户会选择我们的判断层

## 适合访谈的人

- 最近 `90` 天内部署过开源项目的人
- 小团队里的兼职技术负责人
- 正在筛 GitHub 项目是否值得部署的人

## 访谈记录表头

```csv
date_interviewed,person_label,persona,last_project,last_project_link_or_type,scenario,main_goal,main_uncertainty,feared_outcome,current_workaround,ai_tools_used,ai_helped_where,ai_failed_where,other_sources_used,what_felt_fragmented,what_needed_ordering,would_continue_with_checklist,would_send_project_for_review,would_pay_for_deeper_help,preferred_format,quote_best,notes
```

## 访谈问题清单

### A. 还原最近一次真实场景

1. 你最近一次差点部署或已经部署的是什么项目？
2. 当时你想达成的结果是什么？
3. 你最不确定的是什么？
4. 你最怕发生什么后果？

### B. 当前怎么解决

5. 你当时先去哪里找答案？
6. 你看了 README、docs、issue、博客、视频还是问人？
7. 这些信息里哪一部分最有用？
8. 哪一部分最碎、最烦、最让你没有把握？

### C. AI 使用情况

9. 你有没有用 ChatGPT、Claude、Gemini、Perplexity 之类的 AI？
10. 你给 AI 的输入大概是什么？
11. AI 帮到了你哪一步？
12. AI 最不够的地方是什么？

### D. 对“判断层”的需求

13. 如果有人帮你先判断“值不值得部署 + 先做什么 + 哪些风险必须先收住”，这对你有价值吗？
14. 你更想要：
- 一份 checklist
- 一份 SOP
- 一个项目级判断结果
- 一次更完整的服务
15. 你愿不愿意把一个真实项目链接发出来让人先看一眼？
16. 你愿不愿意为更完整的判断或顺序付费？
17. 什么情况下，你会觉得 README、issue、AI、朋友或顾问都不够，需要一个外部判断层？
18. 如果要你选择我们，最重要的理由必须是什么：更准、更有顺序、更可信、更便宜、更快，还是更容易复用？
19. 你更愿意先为什么付费：一份数字资产、一次项目判断、还是一段完整上线陪跑？

## 记录要求

每次访谈后至少补这 8 项：

- `quote_best`
  - 最像用户原话的一句
- `current_workaround`
- `ai_failed_where`
- `why_not_current_solution`
  - 为什么当前替代方案不够
- `why_us_now`
  - 为什么这个时刻可能需要我们的判断层
- `consensus_stage`
  - 场景 / 代价 / 替代方案缺口 / 判断层 / 下一步
- `preferred_format`
- `preferred_business_model`
  - 数字资产 / 项目判断服务 / 完整服务模块 / unclear
- `would_pay_for_deeper_help`

## 最低完成标准

- `10-15` 个访谈
- 至少 `5` 个访谈明确记录 AI 使用情况
- 至少 `5` 个访谈明确表达当前替代方案哪里不够
- 至少 `3` 个访谈明确表达更偏好哪种产品形态
- 至少 `3` 个访谈能说出“为什么会选择我们而不是继续用现有方案”的具体理由

## 目标

访谈结束后，你应该能回答：

- 用户最怕什么
- 当前最常见 workaround 是什么
- AI 最常卡在哪一步
- 客户为什么必须选我们，或者为什么暂时没有理由选我们
- 用户更想要数字资产、项目判断，还是更完整服务
