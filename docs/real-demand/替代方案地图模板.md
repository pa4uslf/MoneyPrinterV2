# 替代方案地图模板

日期：`2026-04-20`

这份模板用于研究目标用户当前到底靠什么解决问题。

注意：

- 不要只记录“和我类似的公司”
- 要记录所有现实中的替代方案
- 包括 README、issue、朋友、AI、顾问、托管平台、课程、模板，甚至“先不部署”

## 使用方式

每个替代方案占一行。

优先按下面 6 类补齐：

1. 免费静态替代
2. 免费互动替代
3. 通用 AI 替代
4. 专业服务替代
5. 数字资产替代
6. 放弃型替代

## 表头

```csv
date_logged,name,type,url_or_entry_point,target_user,use_moment,core_promise,format,cta,price_or_cost,proof_type,common_complaints,why_users_choose_it,why_users_leave_it,our_possible_response,notes
```

## 字段说明

- `date_logged`
  - 记录日期
- `name`
  - 替代方案名称
- `type`
  - 允许 starter values:
    - `readme_or_docs`
    - `issue_or_discussion`
    - `blog_or_video`
    - `general_ai`
    - `consultant_or_service`
    - `managed_service`
    - `template_or_course`
    - `friend_or_community`
    - `do_nothing`
- `url_or_entry_point`
  - 链接、产品页、仓库、视频或进入点说明
- `target_user`
  - 这个替代方案主要服务谁
- `use_moment`
  - 用户会在什么时刻用它
- `core_promise`
  - 它在用户心里承诺解决什么
- `format`
  - 内容、工具、服务、课程、问答、顾问等
- `cta`
  - 它要求用户采取什么动作
- `price_or_cost`
  - 金钱成本，或时间成本
- `proof_type`
  - 它靠什么建立信任
  - 例如：stars、SEO、案例、个人品牌、评论数
- `common_complaints`
  - 用户常见抱怨
- `why_users_choose_it`
  - 用户为什么先选它
- `why_users_leave_it`
  - 用户为什么觉得它不够
- `our_possible_response`
  - 我们可能如何差异化应对
- `notes`
  - 其他备注

## 示例行

```csv
2026-04-20,ChatGPT,general_ai,https://chatgpt.com,独立开发者,准备部署前快速问方案,"快速得到部署/安全建议",chat,继续追问,免费或订阅费,"通用品牌信任+即时回答","太泛, 没项目上下文, 没顺序",快, 便宜, 随手可用,"关键决策不敢全信, 不够项目级","强调 go/no-go、top risks、first 5 steps、证据引用",核心替代方案
```

## 最低完成标准

- 至少 `15` 个条目
- 至少 `3` 个通用 AI 条目
- 至少 `3` 个 GitHub / 社区条目
- 至少 `3` 个博客 / 视频条目
- 至少 `2` 个服务 / 托管条目
- 至少 `2` 个数字资产条目

## 输出目标

最终你应该能回答：

- 用户最常先用什么替代方案
- 哪些替代方案最容易让人不满意
- 我们最不该硬碰的优势是什么
- 我们最应该打的短板是什么
