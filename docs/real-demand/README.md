# MoneyPrinter 真需求指引

更新日期：`2026-05-24`

这个目录不是重复已有 roadmap。

它的职责是把《真需求》的判断框架，压到 `MoneyPrinterV2` 当前阶段，回答三个问题：

1. 这个项目当前方向到底靠不靠谱
2. 哪些是真需求，哪些只是自我感动
3. 接下来 90 天应该怎么推进，才不会把“印钞机”做成“印废纸机”

## 先读什么

如果你只想快速判断方向是否正确，先读：

- `2026-04-15-MoneyPrinter方向判断.md`

如果你要继续选题、写资产、做转化和验证，接着读：

- `真实需求识别与验证SOP.md`

如果你要按阶段推进执行，最后读：

- `2026-04-15-90天推进路线图.md`

如果你要开始真正记证据，不要只看模板，直接打开：

- `demand-evidence-samples.csv`
- `substitute-solution-map.csv`

如果你开始担心“这个方向是不是 AI 基于已有内容二次推出来的假设”“用户是不是已经被 README / issue / AI 充分满足”，继续读：

- `2026-04-20-方向归零验证方案.md`

如果你已经找到一些需求，但需要判断“为什么客户必须选我们、如何形成共识、这件事到底适合哪种商业 / 盈利模式”，继续读：

- `价值共识与商业模式假设.md`

如果你担心当前执行陷在已有资产里、没有真正去找市场，继续读：

- `2026-05-24-市场发现冲刺计划.md`

如果你准备开始真正采证，继续用下面这些模板：

## 这组文档解决什么问题

它主要解决四个当前风险：

- 项目战略方向基本成型，但真实外部证据还偏少
- 仓库里“文档进展”明显快于“需求验证进展”
- 目标人群现在还偏宽，容易把内容写成泛建议
- 旧的 `MoneyPrinter` 自动化外壳还在，容易让执行重心滑回“多发内容”而不是“更懂需求”

## 当前总判断

一句话：

> 当前方向大体靠谱，但还处在“方向已对、证据不足、载体偏旧”的阶段。

所以当前优先级不是再扩大战线，而是：

1. 缩窄真实用户场景
2. 补足真实需求样本
3. 验证 checklist 漏斗
4. 只有在重复证据出现后，才推进 SOP 和 `shop.fzhang.dev`

## 当前硬规则

- 没有重复出现的真实问题前，不要把 SOP 当成既定成功品
- 没有真实点击、领取、回复、追问前，不要急着扩更多主题簇
- 没有证明用户愿意继续走下去前，不要把重点切到支付、商城、复杂自动化
- 不要把“文档写得清楚”误当成“市场已经认可”
- 不要把“我能想到很多内容”误当成“用户真的需要这些内容”
- 不要把手里已有的 checklist、SOP、文章或 AI workflow 强行推给客户；先证明客户现有替代方案为什么不够，以及为什么这个时刻需要我们的判断层
- 下一阶段先做市场发现冲刺：至少收集 30 条外部需求样本，再决定是否继续 checklist / SOP / shop 路线
