# MoneyPrinter 真需求指引

更新日期：`2026-04-20`

这个目录不是重复已有 roadmap。

它的职责是把《真需求》的判断框架，压到 `MoneyPrinterV2` 当前阶段，回答三个问题：

1. 这个项目当前方向到底靠不靠谱
2. 哪些是真需求，哪些只是自我感动
3. 接下来 90 天应该怎么推进，才不会把“印钞机”做成“印废纸机”

## 先读什么

如果你只想快速判断方向是否正确，先读：

- `2026-04-15-MoneyPrinter方向判断.md`

- `2026-04-20-方向归零验证方案.md`

如果你准备开始真正采证，继续用下面这些模板：

## 这组文档解决什么问题

它主要解决四个当前风险：

- 项目战略方向基本成型，但真实外部证据还偏少
- 仓库里“文档进展”明显快于“需求验证进展”
- 目标人群现在还偏宽，容易把内容写成泛建议
- 旧的 `MoneyPrinter` 自动化外壳还在，容易让执行重心滑回“多发内容”而不是“更懂需求”

## 当前总判断

一句话：

> 当前方向大体靠谱，但还处在“方向已对、证据不足、载体偏旧”的阶段。

所以当前优先级不是再扩大战线，而是：

1. 缩窄真实用户场景
2. 补足真实需求样本
3. 验证 checklist 漏斗
4. 只有在重复证据出现后，才推进 SOP 和 `shop.fzhang.dev`

## 当前硬规则

- 没有重复出现的真实问题前，不要把 SOP 当成既定成功品
- 没有真实点击、领取、回复、追问前，不要急着扩更多主题簇
- 没有证明用户愿意继续走下去前，不要把重点切到支付、商城、复杂自动化
- 不要把“文档写得清楚”误当成“市场已经认可”
- 不要把“我能想到很多内容”误当成“用户真的需要这些内容”
