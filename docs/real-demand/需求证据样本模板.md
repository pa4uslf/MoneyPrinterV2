# 需求证据样本模板

日期：`2026-04-20`

这份模板用于沉淀每一条真实需求样本。

它不是漏斗数据表。

它负责记录：

- 用户原话
- 场景
- 恐惧
- 当前替代方案
- AI 是否已被使用
- 后续是否愿意继续走

## 表头

```csv
sample_date,channel,source_url_or_person,persona_guess,user_quote,scenario,current_blocker,feared_outcome,current_workaround,current_ai_tools_used,current_ai_limitations,other_sources_used,requested_next_step,willing_to_pay_or_book,matched_asset,matched_offer_type,notes
```

## 字段说明

- `sample_date`
  - 样本日期
- `channel`
  - 例如：
    - `github_issue`
    - `github_discussion`
    - `reddit`
    - `v2ex`
    - `zhihu`
    - `private_chat`
    - `email_reply`
    - `interview`
- `source_url_or_person`
  - 来源链接或匿名对象标识
- `persona_guess`
  - 你推测的用户类型
- `user_quote`
  - 尽量保留原话
- `scenario`
  - 用户处在什么具体场景
- `current_blocker`
  - 当前卡点
- `feared_outcome`
  - 最怕什么后果
- `current_workaround`
  - 当前主要怎么解决
- `current_ai_tools_used`
  - 用了什么 AI
- `current_ai_limitations`
  - AI 卡在哪里
- `other_sources_used`
  - README / issue / docs / blog / friend 等
- `requested_next_step`
  - 用户想继续要什么
- `willing_to_pay_or_book`
  - 是否出现愿意付费、预约、发项目链接等信号
- `matched_asset`
  - 更适合匹配哪篇文章 / checklist / SOP
- `matched_offer_type`
  - 允许 starter values:
    - `content_only`
    - `checklist`
    - `sop`
    - `project_judgment`
    - `broader_service`
- `notes`
  - 备注

## 示例行

```csv
2026-04-20,github_discussion,https://github.com/example/repo/discussions/123,indie_dev,"I can run it locally but I don't know if it's safe to expose publicly.",准备把项目上到 VPS,不知道先收哪些风险,上线后暴露密钥或端口,README + ChatGPT,ChatGPT,"建议太泛, 没有顺序, 不敢明确说能不能上",README+issue,"想要一个 go/no-go 判断",possible,部署前检查清单,project_judgment,高意图样本
```

## 最低完成标准

- `100` 条样本
- 至少 `30` 条明确记录 AI 使用情况
- 至少 `30` 条明确记录当前 workaround
- 至少 `20` 条明确记录是否出现继续走或付费信号

## 目标

最后要能回答：

- 这个问题是否反复出现
- 用户最怕什么
- AI 是否已经足够
- 他们更想要 checklist、SOP、项目判断，还是更完整服务
